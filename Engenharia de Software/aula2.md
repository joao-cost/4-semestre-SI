# O que é Engenharia de Software?

- É uma área da Computação que aplica princípios, metódos e ferramentas para o desenvolvimento, manutenção e evolução, buscando qualidade, eficiência e confiabilidade.

## Por que Engenharia de Software é importante?

- Reduz custos de manutenção;
- Aumenta qualidade de produto final;
- Facilita escabilidade e volução do sistema;
- Minimiza erro e falhas durante o desenovlvimento;
- Organiza o trabalho em equipe.

# Diferença entre Programação e Engenharia de Software

| Programação                          | Engenharia de Software                |
|--------------------------------------|---------------------------------------|
| Foco na condificação.                | Foco em todo o ciclo de vida do software. |
| Resolve problemas pontuais.          | Garante qualidade, manutenção e evolução. |
| Pode ser feita de forma improvisada. | Segue métodos e processos.                |

# Principais atividades de Engenharia de Software
 - Análise de requisitos; -> Levantamento das necessidades do cliente.
 - Projeto de software; -> Definição da arquitetura e madelagem.
 - Implementação; -> Codificação do sistema.
 - Testes; -> Verificação e validação.
 - Implantação; -> COlocar o sitema em produção.
 - Manutenção. -> Correção e atualização do software.

# Processo de Software
- Conjunto de atividades, métodos, ferramentas e práticas usadas para desenvolver e manter um software, desde a ideia inicial a sua retirada de uso.
- Defini as estapas para construir um sistema.

## Fases genéricas
    - Definição
        - **O que deve ser feito?**
            - Quais informações são importantes;
            - Função e desenpenho desejados;
            - Tipos de interface;
            - Restrições de projeto;
            - Critério de validação.
        - **Como fazer?**
            - Definição das estruturas de dados;
            - Arquitetura de software;
            - Procedimentos;
            - Como os testes serão realizados;
            - Como o projeto será traduzido para uma linguagem de programação.
    - Desenvolvimento

    - Manutenção
        - **Quais mudanças?**
            -  Concentra-se nas mudanças realizadas no software após a sua implantação.
            - *Tipos:*
                - Corretiva: modifica o software para corrigir defeitos;
                - Adaptativa: modifica o software para acomodar mudanças no ambiente externo;
                - Preventiva: aprimora o software além dos requisitos funcionais originais.
                - Perfectiva: faz modificações de modo que o programa possa ser mais facilmente corrigido, adaptado e melhorado.

## Processo
Atividades -> Ações -> Tarefas -> Software de Qualidade
| Buscam atingir um objetivo amplo, independente da complexidade do projeto | Envolvem um conjunto de tarefas e resultam em um artefato de software | Concentram-se em um objetivo pequeno, porém bem definido |

## Pontos Importantes
- Os diferente processos de software organizam as atividades de maneiras diversas e com diferentes níveis de detalhes;
- Os prazos das atividades veriam, do mesmo modo que os resultados de cada atividade;
- Diferentes organizações podem utilizar processos diferentes para produzir o meso tipo de produto;
- Alguns processos são mais adequados do que outros para alguns tipos de aplicação;

### Objetivos do processo
- Garantir qualidade no produto final.
- Tornar o desenvolvimento previsível e organizado.
- Facilitar a comunicação entre membros da equipe.
- Morarar a produtividade e reduzir custos.
- Garantir que os prazos sejam cumpridos.

# Modelos de Processo de Software
## Modelos Prescritivos
- Prescrevem um conjunto de elementos de processos;
- Propõem um fluxo de processos para os elementos;
- Acomodam as fases genéricas de modos específicos.
- Fornecem um roteiro eficaz para a equipe.

Cascata -> Incremental -> Prototipação -> Esperial

### Modelo Cascata
- Facíl entendimento;
- Útil em escopo com pouco complexidade.
- Eficientes quando os requisitos estão bem definidos;
- Útil para adequaççao em um sistema existente.

### Modelo Incremental
- Menor custo para acomodar mudanças nos requisitos.
- Útil em situações de poucos recursos humanos.
- Os incrementos servem para avaliação do usuário.
- Ajuda a administrar riscos técnicos.

### Modelo Prototipação
- Prototipação é uma técnica que consiste em criar um modelo ou protótipo do sistema antes de sua implementação final.
- Esclarece requisitos;
- Auxilia na compreensão do problema.
- Usuário pode ter uma prévia;
- O Produto final é mais coerente com os requisitos.
- Menos chances de retabalho.

### Modelo Expiral
- Junção do Modelo Cascata com o Prototipação.
- É uma abordagem realista;
- Melhora a compreensão e reação aos riscos.
- Reduz riscos;
